🚖 نظام إدارة التاكسي الذكي | Smart Taxi Management System (STMS)
النسخة المستقرة: v1.1.2

نظام سحابي متكامل يعتمد على تقنيات Supabase لإدارة أسطول التاكسي، العُهد المالية، والمطابقة الدقيقة بين الملاك والسائقين.

📋 نظرة عامة (Project Overview)
تم تصميم هذا النظام ليكون الحل الأمثل لإدارة العمليات اليومية لشركات التاكسي، حيث يغطي الدورة المالية كاملة من لحظة استلام الإيرادات من السائقين وحتى توزيع الأرباح على الملاك، مع نظام تتبع دقيق للمصاريف والحالات التشغيلية.

🛠 التقنيات المستخدمة (Tech Stack)
Frontend: HTML5, CSS3 (Custom Design System), JavaScript (Vanilla ES6).

Backend/Database: Supabase (PostgreSQL + Real-time).

Authentication: Row Level Security (RLS) via Supabase.

Architecture: نظام ملفات منفصل (3-File Protocol) لكل موديول.

🏗 هيكلية المشروع (File Structure)
يتبع المشروع "دستور معايير مشروع MSP" الصارم:

global-config.js: المحرك المركزي (مفاتيح Supabase، القائمة الجانبية، الساعة الرقمية).

index.html / dashboard.js: لوحة التحكم الذكية وفلاتر المحصلين.

revenues.html / revenues.js: إدارة الإيرادات اليومية.

expenses.html / expenses.js: إدارة المصاريف التشغيلية وصيانة الأسطول.

dashboard.css: نظام التنسيق الموحد (Luxury UI).

💎 المميزات الرئيسية (Core Features)
1. لوحة التحكم الذكية (Smart Dashboard)
فلترة البيانات حسب الشهر و المحصل.

بطاقات ديناميكية لأصحاب السيارات تظهر صافي الدخل لكل سيارة تحت إدارتهم.

تحليل إنتاجية السائقين وأداء الأسطول بشكل لحظي.

2. إدارة الإيرادات والمصاريف (Financial Core)
نظام Foreign Key صارم يمنع تداخل البيانات.

تسجيل المصاريف مع ربطها بالموردين والموظفين المسؤولين.

واجهات إدخال ذكية تمنع الأخطاء البشرية.

3. دستور المعايير (Standard Protocol)
كل صفحة تتكون من 3 ملفات منفصلة.

نظام ترقيم إصدارات تصاعدي (Version Control).

تعليقات برمجية توضح المدخلات والمخرجات (Informational Blocks).

📜 قواعد البيانات (Database Schema - Golden Rules)
الجداول: تبدأ بحرف t (مثال: t01_cars, t05_revenues).

الحقول: تبدأ بحرف f (مثال: f02_plate_no).

العلاقات: ربط رقمي (BigInt) لضمان سرعة الاستعلامات وصحة البيانات.

🚀 كيفية التشغيل (Deployment)
قم برفع الملفات على أي استضافة (Static Hosting) أو تشغيلها عبر Live Server.

تأكد من إعداد مفاتيح Supabase في ملف global-config.js.

النظام يدعم التصفح من كافة الأجهزة (Responsive Design).
